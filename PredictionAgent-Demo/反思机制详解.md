## 1. 什么是反思机制

### 定义

**反思（Reflection）** 是一种让AI系统在执行过程中主动检查、评估和修正自己行为的机制。

### 核心思想

```
执行 → 评估 → 决策 → 修正/重试 → 继续
```

Agent不仅执行任务，还要在每一步后问自己：
- "我做对了吗？"
- "结果质量如何？"
- "需要重试吗？"
- "如何改进？"

---

## 2. 为什么需要反思

### 问题场景

| 场景 | 无反思 | 有反思 |
|------|--------|--------|
| 产品识别错误 | 返回错误结果 | 发现置信度低，尝试其他方案 |
| 数据获取失败 | 返回空数据 | 自动回退到模拟数据 |
| 图表生成失败 | 程序崩溃 | 跳过或重试 |
| 分析结果不完整 | 返回不完整结果 | 重试或要求用户澄清 |

### 反思的价值

1. **提高准确性**: 通过验证避免错误结果
2. **增强鲁棒性**: 自动处理异常和失败
3. **自适应**: 根据反馈调整行为
4. **可追溯**: 完整的执行历史记录

## 4. 核心组件

### 4.1 反思状态 (ReflectionState)

```python
@dataclass
class ReflectionState:
    enabled: bool = True                    # 是否启用
    reflection_count: int = 0              # 反思次数
    max_reflections: int = 5               # 最大反思次数

    records: List[ReflectionRecord] = []   # 执行记录
    retry_strategy: RetryStrategy = ...    # 重试策略
    reflection_summary: List[str] = []     # 反思摘要
```

### 4.2 验证结果 (ValidationResult)

```python
@dataclass
class ValidationResult:
    is_valid: bool = True                  # 是否通过验证
    score: float = 1.0                    # 质量分数 0-1
    error_type: ErrorType = None          # 错误类型
    error_message: str = ""               # 错误消息
    suggestions: List[str] = []           # 改进建议
```

### 4.3 错误类型 (ErrorType)

```python
class ErrorType(Enum):
    # 产品识别错误
    PRODUCT_NOT_FOUND = "product_not_found"
    PRODUCT_AMBIGUOUS = "product_ambiguous"
    PRODUCT_MISIDENTIFIED = "product_misidentified"

    # 数据获取错误
    DATA_NOT_FOUND = "data_not_found"
    DATA_INSUFFICIENT = "data_insufficient"

    # 图表生成错误
    CHART_GENERATION_FAILED = "chart_generation_failed"

    # 分析错误
    ANALYSIS_INVALID = "analysis_invalid"
    ANALYSIS_INCOMPLETE = "analysis_incomplete"
```

### 4.4 重试动作 (RetryAction)

```python
class RetryAction(Enum):
    REIDENTIFY_PRODUCT = "reidentify_product"    # 重新识别产品
    REFETCH_DATA = "refetch_data"              # 重新获取数据
    REGENERATE_CHART = "regenerate_chart"        # 重新生成图表
    REANALYZE = "reanalyze"                    # 重新分析
    USE_FALLBACK = "use_fallback"              # 使用备选方案
    ASK_USER = "ask_user"                      # 询问用户
    SKIP_STEP = "skip_step"                    # 跳过步骤
```

### 4.5 评估器 (ResultEvaluator)

评估各步骤的输出质量：

```python
class ResultEvaluator:
    def evaluate(self, step, input_data, output_data) -> ValidationResult:
        """
        评估步骤输出
        """
        if step == "product_identification":
            return self._evaluate_product_identification(...)
        elif step == "data_fetch":
            return self._evaluate_data_fetch(...)
        # ...
```

### 4.6 修正器 (ResultRevisor)

根据评估结果决定修正动作：

```python
class ResultRevisor:
    def decide_action(self, step, validation, retry_strategy) -> (bool, RetryAction):
        """
        决定是否重试以及采取什么动作
        """
        if validation.is_valid and validation.score >= 0.7:
            return False, RetryAction.SKIP_STEP  # 通过，继续

        # 根据错误类型选择动作
        if validation.error_type == ErrorType.PRODUCT_NOT_FOUND:
            return True, RetryAction.REIDENTIFY_PRODUCT  # 重试
        # ...
```

## 5. 执行流程

### 带反思的执行流程

```
开始
  │
  ▼
┌─────────────────────┐
│  执行步骤N           │
│  ProductIdentification / DataFetch / Chart / Analysis
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│  评估结果            │
│  ResultEvaluator     │
└──────────┬──────────┘
           │
           ▼
     ┌─────┴─────┐
     │ 评估通过？ │
     └─────┬─────┘
      Yes  │  No
       │   │
       │   ▼
       │  ┌─────────────────────┐
       │  │ 决定动作            │
       │  │ ResultRevisor      │
       │  └──────────┬──────────┘
       │             │
       │    ┌────────┼────────┐
       │    ▼        ▼        ▼
       │  重试    回退    询问用户
       │    │        │        │
       │    └────────┴────────┘
       │             │
       ▼             ▼
┌─────────────────────┐
│  记录反思           │
│  ReflectionState    │
└──────────┬──────────┘
           │
           ▼
     ┌─────┴─────┐
     │ 全部完成？ │
     └─────┬─────┘
      Yes  │  No
       │   │
       ▼   └──→ 返回"执行步骤N"
    结束
```

### 具体示例：产品识别失败

```
用户查询: "分析XYZ12345产品"

Step 1: 产品识别
├─ 执行: 调用LLM识别产品
├─ 结果: {"identified": false, "confidence": 0.2}
├─ 评估: ValidationResult(is_valid=false, score=0.2)
│        error_type=PRODUCT_NOT_FOUND
│        suggestions=["提供更具体的产品名称"]
│
├─ 决策: 可以重试，选择 REIDENTIFY_PRODUCT
├─ 修正: 添加提示词建议
│
├─ 重试2: 扩大搜索范围
├─ 结果: {"identified": true, "confidence": 0.7, alternatives=[...]}
├─ 评估: ValidationResult(is_valid=true, score=0.7)
│
└─ 通过: 继续下一步
```

## 6. 错误处理

### 错误处理策略

| 错误类型 | 错误处理 | 回退方案 |
|----------|----------|----------|
| 产品未找到 | 重新识别 | 询问用户 |
| 数据不足 | 扩展时间范围 | 使用模拟数据 |
| 数据无效 | 重新查询 | 使用模拟数据 |
| 图表失败 | 重新生成 | 跳过图表 |
| 分析不完整 | 重新分析 | 返回原始数据 |

### 重试策略配置

```python
retry_strategy = RetryStrategy(
    max_retries=3,
    backoff_multiplier=2.0,    # 指数退避
    initial_delay=1.0,          # 初始延迟1秒

    step_configs={
        "product_identification": {
            "max_retries": 2,
            "fallback_action": RetryAction.ASK_USER
        },
        "data_fetch": {
            "max_retries": 3,
            "fallback_action": RetryAction.USE_FALLBACK
        },
        "chart_generation": {
            "max_retries": 2,
            "fallback_action": RetryAction.SKIP_STEP
        },
        "analysis": {
            "max_retries": 2,
            "fallback_action": RetryAction.REFETCH_DATA
        }
    }
)
```

## 7. 使用示例

### 7.1 基础使用

```python
from src.agent_reflective import create_reflective_agent

# 创建反思型Agent
agent = create_reflective_agent()

# 执行分析（自动反思）
result = agent.analyze(
    query="分析iPhone销量预测",
    chart_type="combined",
    use_mock_data=True,
    enable_reflection=True  # 启用反思
)

print(f"成功: {result['success']}")
print(f"反思记录: {result.get('reflection')}")
```

### 7.2 禁用反思

```python
# 快速执行，不需要反思
result = agent.analyze(
    query="快速查询",
    enable_reflection=False  # 禁用反思
)
```

### 7.3 自定义评估

```python
from src.reflection import ResultEvaluator

evaluator = ResultEvaluator()

# 手动评估结果
validation = evaluator.evaluate(
    step="product_identification",
    input_data={"query": "iPhone"},
    output_data={"identified": True, "confidence": 0.9}
)

if not validation.is_valid:
    print(f"需要重试: {validation.suggestions}")
```

### 7.4 查看反思历史

```python
result = agent.analyze(query="分析...", enable_reflection=True)

reflection = result.get("reflection", {})
for record in reflection.get("records", []):
    print(f"步骤: {record['step']}")
    print(f"验证: {record['validation']}")
    print(f"重试次数: {record['retry_count']}")
```

## 8. 最佳实践

### 何时使用反思

| 场景 | 推荐 |
|------|------|
| 关键业务决策 | ✓ 使用反思 |
| 快速原型验证 | ✗ 禁用反思 |
| 用户交互场景 | ✓ 使用反思 |
| 批量处理 | ✗ 禁用反思 |
| 复杂查询 | ✓ 使用反思 |

### 反思配置建议

```python
# 生产环境
reflection_state = ReflectionState(
    enabled=True,
    max_reflections=5,
    retry_strategy=RetryStrategy(max_retries=3)
)

# 开发/调试
reflection_state = ReflectionState(
    enabled=True,
    max_reflections=10,  # 更多反思机会
    retry_strategy=RetryStrategy(max_retries=5)
)

# 快速执行
reflection_state = ReflectionState(enabled=False)
```

### 性能考虑

- 反思会增加执行时间（约20-50%）
- 复杂场景可限制反思次数
- 可在关键步骤启用，旁路步骤禁用

### 监控指标

```python
# 反思效率指标
reflection_rate = total_reflections / total_executions
retry_rate = total_retries / total_steps
success_after_retry = successful_retries / total_retries

# 质量指标
avg_quality_score = sum(scores) / len(scores)
fallback_rate = fallback_count / total_executions
```
